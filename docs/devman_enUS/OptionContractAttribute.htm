OptionContractAttribute must be applied on the class target, which represents an option contract in ACF.

Property

Description

Remarks

Type

Indicates the type of the option contract

 There are four different types: Patternized, Exact, None and Free. For more information about option contract types, please refer to Syntax (Contract) Types chapter.
Argument Indicates the corresponding argument of the option contract When the type of the option contract is Exact, the Argument property stands for the command line argument of the contract. For example, when a console application has only one argument called "/version", which is for displaying the version information, we can create an Exact contract and set the Argument property of the attribute to "/version". Use semicolon ";" to separate the different instance of the argument, like: "/version;/ver", which makes the application had the same behavior when user provides the "/version" or "/ver" argument. You don't need to specify the Argument property when the option contract is the type of Patternized, Free or None.
CaseSensitive Specifies if the Argument is case sensitive. Also this property is meaningful when it is an Exact option contract. It is used for determining whether the Argument is case sensitive. "/Version" and "/version" will have different meaning if this property has been set to True. You don't need to specify this property when the option contract is the type of Patternized, Free or None.
Description Specifies the description of the option contract. This description text will be used for generating the help information that will be shown on the help screen. It is a best practice that specify the Description property for each option contract in your contract repository.
Parameters Specifies the number of the arguments that are the type of Parameter. A console applicationn may have one or more arguments that are the type of Parameter. This property is for setting the exact number of such arguments. In the catool application for example, it just have, and just need two parameters: the two integer numbers. Give the value -1 to this property (or ignore the property) if your parameter list is variant. It is meaningful only when the contract is the Patternized type. You don't need to specify this property when the option contract is the type of Exact, Free or None. For more information about the Parameter type please refer to Parameter Type.
