Until now, we just have two class libraries, one is the application provider, the other is the library which holds two different contracts (we call this library a Contract Repository in ACF, and it will be discussed in more detail in the Contract Repository chapter). None of them is executable. In order to make it work in command line, we must create a console application.

This console application is pretty simple, it just an host for the defined contracts. We will implement this console application with few lines of code.

Before we start, we should add a console application to our solution. Please do the following:

  1. Right click the solution, select Add | New Project
  2. In the Add New Project dialog, select Visual C# | Console Application, in the Name text box, type "catool"
  3. Right click on the References node of the newly created console application, select Add Reference
  4. Under .NET tab of the Add Reference dialog, select Adaptive Console Framework, then click OK button
  5. Right click on the References node of the newly created console application, select Add Reference
  6. Under Projects tab of the Add Reference dialog, select Catool.Contracts and Catool.Provider projects, then click OK button. This ensures that the generated binary files of these two projects will be copied into the console application's binary directory
  7. Open the Properties\AssemblyInfo.cs file, modify the file according to your needs. Following gives the example.

    using System.Reflection;

    using System.Runtime.CompilerServices;

Note: Actually in the step 6 above, it is not necessary to add the project references to the console application, because the console application is independent of the implementation and it don't need to know much about the details of the provider and contracts in ACF architecture, everything is configurable in ACF. Here you can redirect the output of these two projects to the console application's binary directory by project settings, instead of setting the project references.
