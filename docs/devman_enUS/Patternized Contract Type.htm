Patternized contract type is the one that we use most frequently. In a Patternized contract type, we can define the options, the parameters, obtain the option values and define switches.

Some command line syntax includes the options and switches, like /type:library, /output:a.txt,b.txt and /silent. It can also include some parameters with which the application will process. For example, in the catool console application which is provided as an example along with the ACF package, it can print the subtraction result of the two numbers by using the command line like "catool.exe /method:sub 20 15", where 20 and 15 are the parameters with no option names. This kind of command line syntax matches a specific pattern. This is also why we call it Patternized.

In the first example we listed in Syntaxes and Option Contracts topic, all of the syntaxes provided by xsd.exe are the Patternized type. In order to use xsd.exe, users need to provide not only the file name that needs to be processed, but also additional options and switches.

Take the catool.exe as an example, it has the following Patternized contract:

 catool.exe </method|/m:> [/nologo|/n] p1 p2
