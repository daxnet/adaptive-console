OptionAttribute is applied on the properties of the option contract class, it is for getting the value of the corresponding argument when the argument is parsed.

Consider the following code snippet that we have mentioned before:

[

    Option(

        Name="/method;/m",

        Type=OptionType.SingleValue,

        Description="Specifies the calculation method\n"+

                    "\t\tadd: Performs an addition calculation\n"+

                    "\t\tsub: Performs a subtraction calculation\n"+

                    "\t\tmul: Performs a multiplication calculation\n"+

                    "\t\tdiv: Performs a division calculation",

        Required=true)

]

public string Method { get; set; }

The argument (option) here is a single value option, which means user should provide a "/method:value" or "/m:value" argument when using the command line. For ACF, it will use the value of the argument (value here) to populate the Method property.

There is a question here, what type should I take for the property when I apply the OptionAttribute on it? In the above example, it is obvious that we should use a string property for the single value option, because no matter what type the value is, it is better to be stored in the string field. Following table shows the relationship between the option type and its corresponding property type.

Note: Arguments that are the type of Parameter are stored in a System.String field. For more information about Parameter type, please refer to the Parameter Type chapter.

Following is the detailed description and remarks of the properties in the OptionAttribute class.

Property

Description

Remarks

Name Name of the option It has the same meaning with the Argument property of the OptionContractAttribute. It defines the name of the option. Use semicolon ";" to separate the different instance of the names. For example: "/input;/i".
Description Description of the option This description text will be used for generating the help information that will be shown on the help screen. It is a best practice that specify the Description property for each option in your contract.
Type Type of the option ACF supports two categories (four types) of options. For more information about the option type please refer to Argument (Option) Categories and Types chapter.
Required Indicates if the option is required (mandatory) The default value is true, which means users of the application must provide the option in the command line.
CaseSensitive Specifies if the Option is case sensitive For example, if the CaseSensitive property is set to true, "/input" and "/Input" are two different options.
ValueSeparator Specifies the value separator character for the values of an option It just makes sense for ValueList options. For ValueList options, users can use the separator character to separate the values specified. For example, when this property is set to ",", users must use the comma to separate the values in the ValueList option, like "/inputs:a,b,c". When this property is set to ";", users must use the semicolon to separate the values, like "/inputs:a;b;c". The default value of this property is ',' (comma).
Default Specifies the default value when the option is not required (mandatory) When the Required property is set to false, Give the option a default value by using this property.
