Application provider is a role which provides the concrete console application instance in ACF, it provides the detailed information about console application, and determines the how the application behaves. It is easy to create application provider just by creating a new class which inherits from AdaptiveConsole.ConsoleApplicationBase, and implementing the abstract members.

Create a blank solution named catool in Visual Studio 2008

  1. Open Visual Studio 2008
  2. Select File | New | Project menu, this brings up the New Project dialog
  3. In the New Project dialog, select Other Project Types | Visual Studio Solutions | Blank Solution, then type the name "catool" in the Name text box

Create the Provider

  1. In the Solution Explorer, right click the newly created solution, select Add | New Project menu, this will bring up the Add New Project dialog
  2. Select Visual C# | Class Library project so that we can add a new class library project to the solution, in the Name text box, input Catool.Provider
  3. Right click References node of the Catool.Provider project, select Add Reference menu, the Add Reference dialog will appear
  4. In the .NET tab, select Adaptive Console Framework, and click OK
  5. In the Solution Explorer, rename the Class1.cs to CatoolApplication.cs, on the coming confirm dialog, click OK to accept the change
  6. Import the AdaptiveConsole namespace by "using AdaptiveConsole;" statement
  7. Make the CatoolApplication class inherits from ConsoleApplicationBase class:

    public class CatoolApplication : ConsoleApplicationBase

    { }


  8. Override and implement the Description and Logo properties as follows:

    protected override string Description

    {

        get

        {

            return "A demonstration of" + 

                   "Adaptive Console Framework";

        }

    }

     

    protected override string Logo

    {

        get

        {

            return @"catool";

        }

    }
  9. Explicitly implement the constructor which carries a string array as the only argument, and initialize the base class by using this argument

    public CatoolApplication(string[] args) : base(args) { }
  10. After finishing the steps above, our CatoolApplication class should look like this:

    using AdaptiveConsole;

     

    namespace Catool.Provider

    {

        public class CatoolApplication : ConsoleApplicationBase

        {

            protected override string Description

            {

                get

                {

                    return "A demonstration of " +

                           "Adaptive Console Framework";

                }

            }

     

            protected override string Logo

            {

                get

                {

                    return @"catool";

                }

            }

     

            public CatoolApplication(string[] args) : base(args) { }

        }

    }

Now we have successfully defined the application provider. If you wish to customize the display of logo and body information of the help screen, you can protectly override the BannerTemplate and HelpBodyTemplate properties in your provider class. For more information about these two properties and templates, please refer to the Help Screen Customization chapter.
