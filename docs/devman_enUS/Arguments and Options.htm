Argument Types: SingleValue, ValueList, Switch and Parameter

There are four different types of arguments: SingleValue, ValueList, Switch and Parameter. This documentation will discuss these types in the following chapters.

Arguments and Properties

From the developer's point of view, an argument is equivalent to a property in the option contract class. In the option contract class we defined previously, we have two properties named IsGenerateClass and IsGenerateDataset. Each property refers to an argument or option. The option attribute must be applied on these properties so that ACF can understand. We will discuss the OptionAttribute class in the following chapters.
