It is easy to understand the None contract. When a console application carries no command line arguments, it is a None contract. For example, in catool.exe, if you do not provide any command line argument, you call the application simply by its name, it will print a brief description on the screen:

D:\>catool.exe
catool
Copyright (C) 2007-2009 SunnyChen.ORG, all rights reserved.

An example of Adaptive Console Framework

Performs calculation on the two given numbers.
Type catool.exe/? for more information.

Note: Contract repository can just have no more than one contract whose type is None. Otherwise, an InvalidContractException would be raised.
